# Azureでリソース変更履歴を確認するためのKQLナレッジドキュメント

このドキュメントでは、Log Analytics ワークスペースで Kusto Query Language (KQL) を使って、Azure リソースの変更履歴（誰が、いつ、何を変更したか）を確認する方法をわかりやすく解説します。KQLに不慣れな方でも使えるよう、使い方、クエリの読み方・ポイント、よくある調整方法をまとめています。

---

## 前提条件
- Azure portalにログインできること
- Azure の Log Analytics ワークスペースにアクセス権があること

---

## やりたいこと
- 指定期間内に発生したリソースの変更履歴を一覧表示する
- 作業者（ユーザーやマネージドID）で絞り込む
- 一部のリソース（文字列パターンで指定）を除外することで、作業者が認識しているログを除外する。<br>
こうすることで、***「作業者が不要な変更をしていないか」***、という観点に絞って検索することができます。
---

## 実行手順（Azure ポータル）
1. Azure ポータルにログイン
2. 対象の Log Analytics ワークスペースを開く
3. 左メニューから「ログ（Logs）」をクリック
4. 下記のクエリを貼り付けて「実行」をクリック
5. 結果テーブルで変更履歴を確認

---

## クエリ全文

```{r, attr.source='.numberLines'}
let patterns = dynamic([“la-defenderAlert-stg-Spoketest-001"]);// 除外するリソースIDに含まれる文字列を指定(例：(["RG-1","VM1","VM2"]))
let regex = strcat(@"(?i)(", array_strcat(patterns, "|"), @")");  
arg("").resourcecontainerchanges
| union arg("").resourcechanges
| extend 
    changeTime = properties.changeAttributes.timestamp, 
    targetResourceId = tostring(properties.targetResourceId),
    changeType = tostring(properties.changeType),
    changeProperties = properties.changes,
    changeCount = properties.changeAttributes.changesCount,
    changedBy = tostring(properties.changeAttributes.changedBy)
| where changeTime > ago(36h) // 検索時間を指定(例：1d(過去1日)、30m(過去30分)、1h(過去1時間))
| where changedBy in ("alice@contoso.com","b2c3...-app-id")  // 作業者を指定(例：XXXXX@grcore.com（ユーザー）、XXXX-XXXX-XXXX-XXXXXXX（マネージドIDのアプリケーションID）)
| where not(targetResourceId matches regex regex)
| project changeTime, targetResourceId, changeType, changeCount, changeProperties, changedBy
```

## クエリの意味とポイント解説
上記のクエリ全文において、処理の仕組みについて解説します。

### 出力から除外する文字列の指定
以下の2点を変数として格納します。

- let patterns = dynamic([...])
  - 除外したいリソースIDの一部文字列を配列で指定します。
  - 例: ["RG-Prod","VM-App01","network-"] など。複数指定で OR 条件になります。

- let regex = strcat(@"(?i)(", array_strcat(patterns, "|"), @")")
  - patterns 配列を正規表現に変換します。
  - (?i) は大文字小文字を無視するオプションです。

その後「| where not(targetResourceId matches regex regex)」の絞り込みに変数が渡るので、指定の文字列が含まれていることを条件に、出力から除外する結果となります。

### 検索のメイン処理

- arg("").resourcecontainerchanges | union arg("").resourcechanges
  - Azure Resource Graph からの変更データを結合して、サブスクリプションやリソースレベルの変更をまとめて取得します。
  - `resourcecontainerchanges`テーブルはサブスクリプションやリソースグループ、管理グループへの変更を対象とします。
  - `resourcechanges`テーブルは上記以外のリソースへの変更を対象とします。
  - ただし、Azureポリシーへの変更は含まれません。
  - Log Analyticsのクエリ検索では、テーブル名の前に「arg("").」をつけることで、Azure Resource Graphのテーブルを参照するという特性を持っています。<br>
    Azure Resource Graphから直接クエリを叩く場合、 「arg("").」 は不要です。

- extend 〜
  - 取得した JSON 形式のプロパティを、人が読める列に展開します。
  - 今回は、propertiesの列の中にあるプロパティ(properties.XXXXX)を、別の列に外出しすることで、読みやすい形にします。
  - 主な列:
    - changeTime: 変更が発生した時刻
    - targetResourceId: 変更対象のリソースID
    - changeType: 追加、更新、削除などの種別
    - changeProperties: 何が変わったかの詳細（プロパティ差分）
    - changeCount: 変更されたプロパティ数
    - changedBy: 変更作業者（ユーザーUPNやアプリケーションID）

- where changeTime > ago(36h)
  - 直近36時間の変更のみを表示します。
  - 期間は「ago(1d)」「ago(7d)」「ago(2h)」「ago(30m)」などに変更可能。

- where changedBy in (...)
  - 対象のユーザーやマネージドIDで絞り込みます。
  - 例: `("alice@contoso.com","b2c3...-app-id")` と複数指定可能。
  - すべての変更を見たい場合は、この行を削除します。
  - マネージドIDを指定する場合、用途としてポリシーによるリソースの是正が行われた時などを特定する場合に使用します。

- where not(targetResourceId matches regex regex)
  - 上記の出力から除外する文字列の変数で指定したパターンにマッチするリソースIDを結果から除外します。
  - 除外不要なら、この行を削除します。

- project ...
  - 必要な列のみを表示します。列の順序・種類は自由に変更可能。
